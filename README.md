# Python项目打包工具

一个简单易用的Python项目打包工具，可以将您的Python应用快速打包成Windows EXE可执行文件和Android APK安装包。

**作者：MiniMax Agent**  
**版本：2.0.0**  
**发布日期：2025-11-02**

## ✨ 这个工具能做什么

- **一键打包EXE**：将Python程序打包成单个EXE文件，无需安装Python环境即可运行
- **生成APK应用**：将Python应用打包成Android APK安装包，支持在手机上运行
- **自动处理依赖**：自动分析并包含程序所需的所有库文件
- **支持GUI程序**：完美支持Tkinter、PyQt等图形界面程序
- **资源文件打包**：自动包含图标、图片等资源文件

## 🚀 快速使用

### Windows EXE打包（推荐新手）

1. **准备您的Python文件**
   - 将您的Python脚本（如 `main.py`）放在工具目录中

2. **双击运行打包**
   - 双击 `build_exe.bat` 文件
   - 等待打包完成（通常需要1-5分钟）

3. **获取EXE文件**
   - 打包完成后，在 `dist` 文件夹中找到您的EXE文件
   - 可以直接分享给其他人使用

### Android APK打包

1. **安装必要环境**
   - 需要安装Java JDK和Android SDK（首次使用）
   - 工具会自动帮您检查和安装

2. **运行打包脚本**
   - 双击 `build_apk.bat` 文件
   - 首次打包需要较长时间（10-30分钟）

3. **安装APK**
   - 在 `bin` 文件夹中找到APK文件
   - 传输到手机并安装即可使用

## 📋 系统要求

### Windows EXE打包
- **操作系统**：Windows 7 或更高版本
- **Python**：3.7 或更高版本（用于打包过程）
- **内存**：至少 2GB RAM
- **网络**：首次打包需要下载依赖库

### Android APK打包
- **操作系统**：Windows 10 或更高版本
- **Java**：JDK 11 或更高版本
- **Python**：3.8 或更高版本
- **内存**：至少 4GB RAM（编译过程需要大量内存）
- **网络**：稳定的网络连接（下载Android开发工具）

## 💡 使用建议

### 选择合适的打包方式

**选择EXE打包的情况：**
- 您的程序主要在Windows电脑上使用
- 希望程序启动速度快
- 不需要移动端支持

**选择APK打包的情况：**
- 需要在Android手机上运行
- 程序适合触屏操作
- 愿意接受稍大的文件体积

### 优化打包效果

1. **清理代码**：删除调试代码和测试文件
2. **优化图片**：压缩图片和图标文件
3. **精简依赖**：只导入必要的Python库
4. **测试运行**：打包前先确保程序正常运行

## ❓ 常见问题

### Q: 打包后的文件很大怎么办？
A: 
- EXE文件通常50-200MB，APK文件通常100-300MB
- 这是正常现象，因为包含了Python运行环境和所有依赖库
- 可以通过排除不必要的模块来减小体积

### Q: 打包失败怎么办？
A: 
- 检查Python代码是否有语法错误
- 确保所有依赖库都已安装
- 查看错误信息，通常是缺少某个模块

### Q: EXE文件无法在其他电脑运行？
A: 
- EXE文件包含完整环境，可以在任何Windows电脑上运行
- 如果遇到问题，可能需要安装Visual C++运行库

### Q: APK文件无法安装？
A: 
- 确保Android设备允许安装未知来源应用
- 检查设备存储空间是否充足
- 确认Android版本兼容（Android 5.0+）
